description: High-performance native addon for Node.js
---

The NAPI binding provides a native Node.js addon built with `napi-rs`. It serves as an alternative backend for the JS/TS ORM, offering better performance for large datasets and avoiding WASM overhead.

## Installation

```bash
npm install @alizarin/napi
```

Pre-built binaries are included for:
- Linux x64
- macOS ARM64 (Apple Silicon)
- macOS x64
- Windows x64

Pre-built binaries are also available from [GitHub Releases](https://github.com/flaxandteal/alizarin/releases).

## Using as a JS/TS Backend

The NAPI backend is used through the same JS/TS ORM API. Switch backends with:

```typescript
import { setBackend, setNapiModule } from 'alizarin';

// Load the native addon
const napi = require('@alizarin/napi');
setNapiModule(napi);
setBackend('napi');

// Now all ORM operations use native code
const Person = await gm.loadGraph('Person');
const people = await Person.all(); // native speed
```

## Direct NAPI API

The NAPI module also exposes lower-level classes for direct use without the ORM layer:

### Classes

| Class | Description |
|-------|-------------|
| `NapiStaticGraph` | Parse and register graph definitions |
| `NapiStaticResourceRegistry` | Index and query resources with native performance |
| `NapiPrebuildExporter` | Export graphs and resources to a prebuild directory |
| `NapiRdmCache` | Reference data concept cache |

### NapiStaticGraph

```javascript
const graph = NapiStaticGraph.fromJsonString(graphJsonString);
console.log(graph.graphId, graph.name);
graph.register(); // register in global registry
```

### NapiStaticResourceRegistry

```javascript
const registry = new NapiStaticResourceRegistry();
registry.mergeFromResourcesJson(resourcesJson, true);

// Or load from business data bytes (Buffer)
const refs = registry.loadFromBusinessDataBytes(buffer, true);

// Build indexes for fast lookups
const index = registry.getValueToResourcesIndex(graph, nodeId, true);
const filtered = registry.getFilteredTileValues(
  graph, filterNodeId, filterValues, extractNodeId, true, null
);

// Direct access
const resource = registry.getFull(resourceId);
const summary = registry.getSummary(resourceId);
console.log(registry.length);
```

### Prebuild Import/Export

```javascript
// Import all graphs, collections, and resources from a prebuild directory
const result = importPrebuild('/path/to/prebuild', 'http://localhost:8000/');

// Export graphs to a directory
const exporter = new NapiPrebuildExporter();
exporter.exportAllGraphs('/path/to/output');
```

### Module Functions

| Function | Description |
|----------|-------------|
| `buildGraphFromCsvs(graphCsv, nodesCsv, collectionsCsv, rdmNamespace)` | Build graph definition from CSV strings |
| `buildBusinessDataFromCsv(csvData, graphJson, collectionsJson)` | Build resource instances from CSV data |
| `importPrebuild(prebuildDir, baseUri)` | Load graphs and resources from a prebuild directory |
| `extensionCoerce(datatype, value, config)` | Call extension coercion |
| `extensionRenderDisplay(datatype, tileData, language)` | Call extension display renderer |

## When to Use NAPI vs WASM

| | WASM | NAPI |
|-|------|------|
| **Runtime** | Browser + Node.js | Node.js only |
| **Install** | `npm install alizarin` (bundled) | `npm install @alizarin/napi` |
| **Performance** | Good | Better (native) |
| **Memory** | Limited by WASM linear memory | Full system memory |
| **Dataset size** | Small to medium | Any size |
| **Use case** | Frontend, SSG, prototyping | ETL, batch processing, production pipelines |
