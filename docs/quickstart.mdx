To run Alizarin, you need [Arches](https://github.com/archesproject/arches) JSON,
which can be generated with Arches or, in pure Python, with [AORM](https://github.com/flaxandteal/arches-orm).

Alizarin can run via static JSON files, or from an Arches-compatible API.

It needs, unless precached, JSON endpoints for collections, concepts, resources (tiles) and graphs.

## Installation

```bash
npm install alizarin
```

## Quick Start

### Basic Usage

```typescript
import { ArchesClientRemote, GraphManager } from 'alizarin';

// Connect to an Arches instance
const client = new ArchesClientRemote('https://your-arches-instance.org');
const graphManager = new GraphManager(client);

// Load a resource model
const MonumentModel = await graphManager.get('Monument');

// Query all monuments
const monuments = await MonumentModel.all();

// Access monument data
for (const monument of monuments) {
  console.log(await monument.name);
  console.log(await monument.constructionDate);
  console.log(await monument.heritage_status);
}
```

### Working with Static Data

```typescript
import { ArchesClientRemoteStatic, GraphManager } from 'alizarin';

// Load from static JSON exports
const client = new ArchesClientRemoteStatic('/data/exports');
const graphManager = new GraphManager(client);

// Usage is identical to remote client
const PersonModel = await graphManager.get('Person');
```

### Querying Resources

```typescript
const PersonModel = await graphManager.get('Person');

// Query all people
const people = await PersonModel.all();

// Access data through view models
for (const person of people) {
  console.log(await person.forename);
  console.log(await person.surname);
}
```
